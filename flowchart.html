User Flow — Personal Trainer

Autentikasi, manajemen klien, penyusunan jadwal latihan

↓

Alur penggunaan dari sisi personal trainer dimulai dari proses autentikasi, kemudian bercabang ke tiga modul utama: pengelolaan klien, penyusunan jadwal latihan, dan monitoring ringkasan kepatuhan.

User Flow — Client

Notifikasi latihan, meal logging, tracking progress harian

↓

Client menerima notifikasi jadwal latihan dari PT setiap pagi, melaporkan status penyelesaian latihan, dan mencatat asupan makanan sepanjang hari. Sistem menghitung kalori secara otomatis dan memberikan ringkasan di akhir hari.

TDEE Calculator
Total Daily Energy Expenditure dihitung otomatis dari profil klien menggunakan formula Mifflin-St Jeor, dikombinasikan dengan activity level yang dipilih saat onboarding. Nilai ini menjadi baseline target kalori harian.

Alur Notifikasi Harian

Cron job pagi, alert real-time, ringkasan malam

↓

Sistem menjalankan tiga cron job harian: pengiriman jadwal latihan pagi hari, reminder meal log di siang dan malam, serta ringkasan harian untuk PT dan client di akhir hari.

sequenceDiagram
    participant CRON as Cron Scheduler
    participant API as API Server
    participant DB as PostgreSQL
    participant NS as Notification Service
    participant PT as Personal Trainer
    participant CL as Client

    rect rgb(240, 253, 244)
    Note over CRON: Trigger harian 06:00 WIB
    CRON->>API: Request pengiriman jadwal hari ini
    API->>DB: SELECT * FROM workout_schedules WHERE date = today
    DB-->>API: Return daftar jadwal per klien
    loop Setiap klien yang punya jadwal
        API->>NS: Kirim notifikasi latihan
        NS->>CL: Push + WhatsApp — detail jadwal latihan
    end
    end

    rect rgb(239, 246, 255)
    Note over CL: Aktivitas harian klien
    CL->>API: POST /meals — log makanan baru
    API->>DB: INSERT meal_log
    API->>API: Hitung akumulasi kalori hari ini
    alt Total kalori melebihi target harian
        API->>CL: In-app alert — kalori melebihi batas
    end
    CL->>API: PATCH /workouts/:id — tandai selesai atau skip
    API->>DB: UPDATE workout status
    end

    rect rgb(254, 249, 195)
    Note over CRON: Trigger harian 21:00 WIB
    CRON->>API: Request generate ringkasan harian
    API->>DB: Aggregate meal_logs + workout_schedules
    API->>DB: INSERT daily_summary
    API->>NS: Kirim ringkasan ke PT dan Client
    NS->>PT: Laporan kepatuhan seluruh klien hari ini
    NS->>CL: Ringkasan personal — kalori dan status workout
    end

Skema Database

Entity relationship, tabel utama, dan relasi antar data

↓

Skema relasional yang menghubungkan user, profil PT dan client, jadwal latihan, log makanan, ringkasan harian, dan riwayat notifikasi. Menggunakan UUID sebagai primary key untuk seluruh tabel.

erDiagram
    USERS {
        uuid id PK
        string email
        string password_hash
        enum role "PT atau CLIENT"
        timestamp created_at
        timestamp last_login
    }

    PT_PROFILES {
        uuid id PK
        uuid user_id FK
        string name
        string specialization
        string photo_url
        string phone
        string bio
    }

    CLIENT_PROFILES {
        uuid id PK
        uuid user_id FK
        uuid pt_id FK
        string name
        float weight_kg
        float height_cm
        int age
        float target_weight
        int daily_calorie_target
        enum activity_level "SEDENTARY, LIGHT, MODERATE, ACTIVE"
        enum status "ACTIVE, INACTIVE, COMPLETED"
    }

    WORKOUT_SCHEDULES {
        uuid id PK
        uuid pt_id FK
        uuid client_id FK
        date schedule_date
        time schedule_time
        string title
        json exercises "array of name, sets, reps, duration"
        enum status "PENDING, DONE, SKIPPED"
        string skip_reason
        string photo_proof_url
        timestamp completed_at
    }

    MEAL_LOGS {
        uuid id PK
        uuid client_id FK
        date log_date
        enum meal_type "BREAKFAST, LUNCH, DINNER, SNACK"
        string food_name
        float portion_grams
        int calories
        float protein_g
        float carbs_g
        float fat_g
        timestamp logged_at
    }

    DAILY_SUMMARIES {
        uuid id PK
        uuid client_id FK
        date summary_date
        int total_calories
        int calorie_target
        bool workout_completed
        float weight_log
        int streak_count
    }

    NOTIFICATIONS {
        uuid id PK
        uuid sender_id FK
        uuid receiver_id FK
        string title
        string body
        enum channel "PUSH, WHATSAPP, IN_APP"
        bool is_read
        timestamp sent_at
    }

    USERS ||--o| PT_PROFILES : has
    USERS ||--o| CLIENT_PROFILES : has
    PT_PROFILES ||--o{ CLIENT_PROFILES : manages
    PT_PROFILES ||--o{ WORKOUT_SCHEDULES : creates
    CLIENT_PROFILES ||--o{ WORKOUT_SCHEDULES : receives
    CLIENT_PROFILES ||--o{ MEAL_LOGS : logs
    CLIENT_PROFILES ||--o{ DAILY_SUMMARIES : generates
    USERS ||--o{ NOTIFICATIONS : sends
    USERS ||--o{ NOTIFICATIONS : receives

Integrasi WhatsApp Notification

Setup koneksi, template pesan, dan alur pengiriman otomatis

↓

WhatsApp dipilih sebagai channel utama notifikasi karena penetrasi penggunanya di Indonesia sangat tinggi. Integrasi menggunakan WhatsApp Business API melalui provider pihak ketiga (Fonnte/Wablas) untuk mengirim pesan otomatis ke klien berdasarkan jadwal yang sudah ditentukan sistem.

flowchart TD
    subgraph Setup ["Konfigurasi Awal oleh PT"]
        direction TB
        A[PT Buka Pengaturan] --> B[Input Nomor WhatsApp Bisnis]
        B --> C[Verifikasi via OTP]
        C --> D{Verifikasi berhasil?}
        D -->|Gagal| B
        D -->|Berhasil| E[Nomor Terhubung]
        E --> F[Pilih Template Pesan\nJadwal Latihan / Reminder / Ringkasan]
        F --> G[Set Jadwal Pengiriman\n06:00, 12:00, 21:00 WIB]
        G --> H[Konfigurasi Selesai]
    end

    subgraph Auto ["Pengiriman Otomatis Harian"]
        direction TB
        I[Cron Trigger Sesuai Jadwal] --> J[Ambil Data dari Database\nJadwal Latihan / Meal Reminder / Summary]
        J --> K[Generate Pesan dari Template\nNama klien, detail latihan, kalori]
        K --> L[Kirim ke WhatsApp API]
        L --> M{Status pengiriman?}
        M -->|Terkirim| N[Catat Status: DELIVERED]
        M -->|Gagal| O[Retry Maksimal 3x\nInterval 5 menit]
        O --> P{Masih gagal?}
        P -->|Ya| Q[Catat Status: FAILED\nFallback ke Push Notification]
        P -->|Tidak| N
    end

    subgraph Template ["Jenis Pesan Otomatis"]
        direction TB
        T1[Reminder Jadwal Latihan\nDikirim 06:00 pagi]
        T2[Reminder Meal Log\nDikirim 12:00 dan 19:00]
        T3[Ringkasan Harian\nDikirim 21:00 malam]
        T4[Alert Kalori Berlebih\nDikirim real-time]
        T5[Ucapan Milestone\nStreak 7 hari, target tercapai]
    end

    H --> I
    N --> R[Klien Terima Pesan di WhatsApp]
    R --> S{Klien klik link di pesan?}
    S -->|Ya| S1[Buka Aplikasi FitTrack\nLangsung ke halaman terkait]
    S -->|Tidak| S2[Pesan tetap tersimpan\ndi chat WhatsApp]

Catatan Implementasi
Setiap pesan WhatsApp mengandung deep link ke halaman spesifik di aplikasi. Contoh: link di reminder latihan langsung membuka halaman jadwal hari ini, link di reminder meal log langsung membuka form input makanan. Ini meminimalkan friction dan meningkatkan engagement rate.
Jenis PesanWaktu KirimContoh Isi
Reminder Latihan06:00 WIB"Hai Sarah, hari ini jadwal Upper Body dari Coach Rizky. Bench Press 4x12, Shoulder Press 3x10. Tap di sini untuk lihat detail."
Reminder Meal Log12:00, 19:00 WIB"Sudah makan siang? Jangan lupa catat makananmu hari ini. Sisa kalori: 980 kcal."
Ringkasan Harian21:00 WIB"Ringkasan hari ini: Latihan selesai, kalori 1.650/1.800 kcal (92%). Streak: 7 hari."
Alert KaloriReal-time"Kalori hari ini sudah mencapai 1.850/1.800 kcal. Pertimbangkan porsi makan malam yang lebih ringan."
MilestoneEvent-based"Selamat Sarah! Kamu sudah konsisten 30 hari berturut-turut. Berat badan turun 2.1 kg dari awal."
Estimasi Biaya WhatsApp API
Menggunakan Fonnte (provider lokal), biaya sekitar Rp 100-150 per pesan. Untuk PT dengan 20 klien aktif dan 3 pesan/hari, estimasi biaya bulanan sekitar Rp 180.000-270.000. Biaya ini sudah tercover dari selisih margin Plan Pro (Rp 99.000) jika dihitung per-klien.
