4. Notification Flow

Alur notifikasi harian: jadwal pagi, alert kalori, dan ringkasan malam.

sequenceDiagram
    participant CRON as Cron Job
    participant API as API Server
    participant DB as Database
    participant NOTIF as Notification Service
    participant PT as Personal Trainer
    participant CLIENT as Client

    Note over CRON: Setiap hari jam 06:00 WIB
    CRON->>API: Trigger kirim jadwal hari ini
    API->>DB: Query workout_schedules WHERE date = today
    DB-->>API: Return daftar jadwal

    loop Untuk setiap klien
        API->>NOTIF: Kirim notifikasi
        NOTIF->>CLIENT: Jadwal latihan hari ini
    end

    Note over CLIENT: Siang / Malam
    CLIENT->>API: POST /meals — log makanan
    API->>DB: Simpan meal_log
    API->>API: Hitung total kalori hari ini

    alt Kalori melebihi target
        API->>CLIENT: Alert — kalori melebihi batas harian
    end

    Note over CRON: Setiap hari jam 21:00 WIB
    CRON->>API: Trigger generate daily summary
    API->>DB: Aggregate meal_logs + workout status
    API->>DB: Simpan daily_summary
    API->>PT: Ringkasan kepatuhan klien hari ini
    API->>CLIENT: Ringkasan kalori dan workout harian

6. Database Schema

Relasi antar tabel utama dalam sistem FitTrack.

erDiagram
    USERS {
        uuid id PK
        string email
        string password_hash
        enum role
        timestamp created_at
    }

    PT_PROFILES {
        uuid id PK
        uuid user_id FK
        string name
        string specialization
        string phone
    }

    CLIENT_PROFILES {
        uuid id PK
        uuid user_id FK
        uuid pt_id FK
        string name
        float weight_kg
        float target_weight
        int daily_calorie_target
        enum status
    }

    WORKOUT_SCHEDULES {
        uuid id PK
        uuid pt_id FK
        uuid client_id FK
        date schedule_date
        string title
        json exercises
        enum status
    }

    MEAL_LOGS {
        uuid id PK
        uuid client_id FK
        date log_date
        enum meal_type
        string food_name
        int calories
        float protein_g
        float carbs_g
        float fat_g
    }

    USERS ||--o| PT_PROFILES : has
    USERS ||--o| CLIENT_PROFILES : has
    PT_PROFILES ||--o{ CLIENT_PROFILES : manages
    CLIENT_PROFILES ||--o{ WORKOUT_SCHEDULES : receives
    CLIENT_PROFILES ||--o{ MEAL_LOGS : logs
